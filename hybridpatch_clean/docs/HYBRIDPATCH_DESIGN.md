# HybridPatch Design

Status: Phase 0/1 implementation track, opened 2026-07-03.

## Goal

`hybridpatch` is a new canonical experiment method compared only against `fullrewrite`.
It replaces the abandoned C71 / C30+ AP-pure continuation path. It is not AP-pure:
bounded whole-file rewrite is an explicit route and must be reported as route share.

Frozen baselines stay untouched:

- `anchorpatch`
- `anchorpatch_v2`
- `anchorpatch_v2_2`

## Protocol

The model emits one JSON envelope in one primary LLM call:

```json
{"protocol":"hybridpatch/1","plan":{...},"action":{...}}
```

`plan` records task family, writable files, read-only files, target files, and
obligations. `action.route` is one of:

- `local_patch`: exact local replace/delete/insert.
- `bulk_patch`: literal global replacement and literal whole-line deletion. No regex.
- `dsl_rules`: declarative deterministic block copy/distribution rules.
- `bounded_rewrite`: declared whole-file generation for listed writable files.

Repair budget: at most one JSON/schema-only repair call is allowed. Semantic or
validation-gate failures do not repair. A failed step carries the editable
workspace forward unchanged and records failure telemetry.

## R-ENUM

DSL enumeration limits are machine-readable in `hybrid_schema.py`:

- `DSL_MAX_RULES = 16`
- `DSL_MAX_EXPLICIT_IDS = 64`
- `DSL_MAX_EXPANDED_ACTIONS = 128`

If a rule would enumerate beyond these limits, the model must route to
`bounded_rewrite`. Rule-limit violations are route violations and fail the gate.

## Metrics

Main metrics:

- `RS@10`
- `CriticalFailure@10`

`CriticalFailure@10` counts any adjacent backward RS transition that drops by at
least frozen theta, plus any drop to 0, parse error, mismatch, or evaluator error.
Theta is calibrated once on dev-mini:

```text
positive nonzero adjacent backward RS drops across hybridpatch and fullrewrite
theta = smallest value in {0.05, 0.10, 0.15, 0.20} at or above p75
default theta = 0.10 if no positive drops exist
```

Disclosure metrics:

- route share, especially `bounded_rewrite` share
- copied/generated byte ratio
- effective forward modification failure rate
- validation-gate failure rate
- JSON/schema repair rate and extra token/call cost
- no-op forward rate
- forward completion audit: effective modification, target files present, and task
  entities found in changed regions. This audit is disclosure-only and not part
  of the main RS score.

## Split

Split artifact: `analysis/hybrid_split.json`.

- dev: existing `dev20_20260625`
- val: first 20 samples from existing `holdout_reserve_20260625`
- test: next 20 samples from existing `holdout_reserve_20260625`
- unused reserve: remaining 20 samples

The split script reads only registry/split metadata and sample prompt text. It
does not scan old result directories and does not call an API.

## Guardrails

- `hybridpatch` is the only method id. Aliases are forbidden.
- No silent FR fallback. `full_rewrite_fallback` and scoped-FR branches do not
  belong to this method.
- Honesty replay must cover every hybrid route before any number is reported.
- Val/test reporting must fail if a frozen campaign mixes code fingerprints.
- Bounded rewrite rows are legal HybridPatch rows but must be disclosed as
  identity dilution.
- Prompt/protocol iteration is dev-only. Any post-val fingerprint change sends
  the work back to dev.

## Implementation Map

- `hybrid_index.py`: deterministic file and multi-scale block summaries.
- `hybrid_schema.py`: protocol constants, route ids, R-ENUM limits, schema checks.
- `hybrid_prompt.py`: prompt and extraction.
- `hybrid_executor.py`: deterministic executor and `HybridExecLog`.
- `hybrid_gate.py`: validation gate and forward audit.
- `test_hybrid_executor.py`: byte-level executor/gate regression tests.
- `experiment_runner.py`: top-level `hybridpatch` branch.
- `verify_anchorpatch.py`: raw-response replay for hybrid routes.
- `analyze.py`: hybrid paired metrics and route telemetry.
