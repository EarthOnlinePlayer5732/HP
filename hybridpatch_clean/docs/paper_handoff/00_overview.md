## 一句话主张

在长程往返编辑任务中，让 LLM 输出**受约束的结构化编辑指令**、由确定性执行器落盘（未声明的内容逐字节保留），整体优于让 LLM 每轮**全文重写**（FullRewrite）——后者会把没被要求改动的内容一起静默改坏。

## 术语表（正文均有定义，此处速查）

| 术语 | 含义 |
|---|---|
| DELEGATE-52 | Microsoft 的往返中继编辑基准：LLM 对专业文档先做编辑（forward）再撤销（backward），链式 10 轮，用领域评估器对比原始文档打分 |
| RS（Reconstruction Score） | 重建分数，0–1，衡量 backward 后文档与原始文档的一致程度，由各领域专用评估器计算 |
| round trip（RT） | 一次 forward + backward 往返 |
| FullRewrite（FR） | 官方基准的默认方式：每步让 LLM 输出全部文件的完整新内容 |
| HybridPatch（HP） | 本文方法：LLM 输出一个 JSON 信封（计划 + 一条编辑路径），确定性执行器应用 |
| 信封（envelope） | HP 的单次输出结构 `{protocol, plan, action}` |
| 路径（route） | action 的四选一形态：local_patch / bulk_patch / dsl_rules / bounded_rewrite |
| 字节保留 | 未被任何编辑操作声明的内容由执行器原样复制，不经过模型 |
| preservation_violations | 核心不变量计数器：未声明内容被改动的次数，必须恒为 0 |
| kept-context | 某步的编辑指令被全部拒绝时，该步保持输入文档不变并显式记失败（诚实 no-op） |
| repair | 主调用失败后至多一次的修复调用，携带具体错误信息 |
| 诚实门 | 独立校验器：从存储的原始 LLM 响应出发重算每个 RS，与记录值逐一比对，不一致即判定遥测漂移/造假 |
| 配对比较 | HP 与 FR 在同一 seeded 任务序列上逐步配对，消除任务抽样差异 |
